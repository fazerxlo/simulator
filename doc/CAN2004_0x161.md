# CAN2004 Frame 0x161 — BSI_GAUGES (Peugeot 407)

PSA-RE canonical name: `BSI_GAUGES` / `ETAT_BSI_TEMP_NIVEAU`  
Sender: BSI  
Period: 500 ms (real bus); simulator transmits at 100 ms  
Frame length: 7 bytes

---

| Byte (1-idx) | Idx | Bits | Signal | PSA-RE name | Encoding |
|---|---|---|---|---|---|
| 1 | 0 | 7 | OIL_LEVEL_RESTART | REDEMARRAGE_SONDE_HUILE | 1 = restart oil level probe |

---

## Byte 0 — OIL_LEVEL_RESTART flag

Bit 7 is a one-shot restart command for the oil-level probe. It is normally `0`.
When set by the BSI it instructs the oil-level sensor to reinitialise.

The simulator always encodes byte 0 as `0x00`.

## Oil Temperature (byte 2, idx 2)

**Formula:** `oil_temp_°C = raw − 40`  
**Resolution:** 1 °C  
**Invalid:** `0xFF`  
**Range:** −40 °C (`0x00`) to 214 °C (`0xFE`)

| Example | Raw | Decoded |
|---------|-----|---------|
| Cold (just started) | `0x0A` | −30 °C |
| Warm idle | `0x78` | 80 °C |
| Normal | `0x8B` | 99 °C |
| Invalid | `0xFF` | — |

```python
# Decode
raw = data[2]
if raw == 0xFF:
    oil_temp = None
else:
    oil_temp = raw - 40

# Encode
if oil_temp is None:
    raw = 0xFF
else:
    raw = int(oil_temp + 40)  # clamp 0x00–0xFE
```

## Fuel Level (byte 3, idx 3)

**Units:** percent (0–100 %)  
**Resolution:** 1 %  
**Invalid:** `0xFF`

The fuel level is a direct percentage, not a resistance value. `0` is empty,
`100` is full.

| Example | Raw | Decoded |
|---------|-----|---------|
| Empty / low | `0x05` | 5 % |
| Half tank | `0x32` | 50 % |
| Full | `0x64` | 100 % |
| Invalid | `0xFF` | — |

```python
# Decode
raw = data[3]
if raw == 0xFF:
    fuel_pct = None
else:
    fuel_pct = raw  # already in percent

# Encode
raw = int(fuel_pct) if fuel_pct is not None else 0xFF
```

## Oil Level (byte 6, idx 6)

**Units:** percent (0–100 %)  
**Resolution:** 1 %  
**Invalid:** `0xFF`

This signal was not documented in the original workspace notes. PSA-RE confirms
it is the engine oil *level* (dipstick equivalent), distinct from the *temperature*
in byte 2 (idx 2).

`0xFF` means the probe is not ready or the value is unavailable (e.g. engine still
running, car on a slope, or probe not fitted). The simulator default is `0xFF`
(unavailable).

| Example | Raw | Decoded |
|---------|-----|---------|
| Below minimum | `0x00` | 0 % |
| Minimum | `0x1E` | 30 % |
| Optimal | `0x50` | 80 % |
| Maximum | `0x64` | 100 % |
| Not available | `0xFF` | — |

```python
# Decode
raw = data[6]
if raw == 0xFF:
    oil_level = None
else:
    oil_level = raw  # already in percent

# Encode
raw = int(oil_level) if oil_level is not None else 0xFF
```

## Full Frame Decode Snippet

```python
def decode_0x161(data: bytes) -> dict:
    """Decode a BSI_GAUGES (0x161) frame."""
    assert len(data) >= 7

    oil_temp_raw  = data[2]
    fuel_raw      = data[3]
    oil_level_raw = data[6]

    return {
        'oil_restart':  bool((data[0] >> 7) & 1),
        'oil_temp_c':   None if oil_temp_raw  == 0xFF else oil_temp_raw - 40,
        'fuel_pct':     None if fuel_raw      == 0xFF else fuel_raw,
        'oil_level_pct':None if oil_level_raw == 0xFF else oil_level_raw,
    }
```

## Simulator Implementation

State fields driven by this frame:

| `car.bsi` field | Source byte (idx) | Default |
|-----------------|------------------|---------|
| `oil` | byte 3 (idx 2), raw − 40 | `0` |
| `fuel` | byte 4 (idx 3), raw | `0` |
| `oil_level` | byte 7 (idx 6), raw | `0xFF` (invalid) |

Byte 0 (`OIL_LEVEL_RESTART`) is not currently stored as a state field; it is
always transmitted as `0x00`.

### Frame length note

PSA-RE specifies this frame as **7 bytes**. Earlier simulator versions encoded
8 bytes with a trailing `0xFF` padding byte — this is harmless but non-standard.
The current implementation encodes exactly 7 bytes.
