# CAN2004 Frame 0x1A8 — SPEED_CONTROL / GESTION_VITESSE (Peugeot 407)

PSA-RE canonical name: `SPEED_CONTROL` / `GESTION_VITESSE`  
Sender: BSI  
Period: 200 ms  
Frame length: 8 bytes

This frame carries the state of the speed regulator (cruise control) and speed limiter,
the currently set target speed, and a partial trip odometer.

---

## Signal Map

| Byte (1-idx) | Idx | Bits | Signal | PSA-RE name | Encoding |
|---|---|---|---|---|---|
| 1 | 0 | 7-6 | SPEED_CONTROL_TYPE | FONCT_ACT_LVV_RVV | 0=none, 1=regulator, 2=limiter, 3=adaptive CC |
| 1 | 0 | 5-3 | ACTIVE_FUNCTION_STATUS | ETAT_FONCT_LVV_RVV | see table below |
| 1 | 0 | 2 | ACTIVATION_ATTEMPT | TENT_ACT_LVV_RVV | 1 = attempt to activate in progress |
| 1 | 0 | 1 | CONTROL_UNIT | UNITE_CONSIGNE_LVV_RVV | 0 = km/h, 1 = mph |
| 1 | 0 | 0 | (unused) | | |
| 2-3 | 1-2 | 15-0 | SET_SPEED | VIT_CONS_LVV_RVV | uint16 × 0.01 km/h; invalid `0xFFFF` |
| 4-5 | 3-4 | — | (unused) | | sent as `0x00 0x00` |
| 6-8 | 5-7 | 23-0 | ODOMETER_PARTIAL | ODO_PARTIEL | uint24 × 0.001 km; invalid `0xFFFFFF` |

## Byte 0 — Control Flags

```
Bit 7-6  SPEED_CONTROL_TYPE   — type of active function
Bit 5-3  ACTIVE_FUNCTION_STATUS — operational state
Bit 2    ACTIVATION_ATTEMPT   — transient: attempt in progress
Bit 1    CONTROL_UNIT         — speed unit (0=km/h, 1=mph)
Bit 0    (unused)
```

### SPEED_CONTROL_TYPE (bits 7-6)

| Value | Name | Meaning |
|-------|------|---------|
| `0` | NONE | No speed control function active |
| `1` | REGULATOR | Cruise control (speed regulator) |
| `2` | LIMITER | Speed limiter |
| `3` | ADAPTIVE | Adaptive cruise control (ACC/radar) |

### ACTIVE_FUNCTION_STATUS (bits 5-3)

| Value | Name | Meaning |
|-------|------|---------|
| `0` | STANDBY | Function available but not active |
| `1` | ACTIVE | Regulator or limiter actively controlling |
| `2` | LIMITER_ACTIVE | Limiter phase of adaptive CC |
| `3` | OVERSPEED_NO_PEDAL | Set speed exceeded, driver not pressing accelerator |
| `4` | OVERSPEED_PEDAL | Set speed exceeded, driver pressing accelerator |
| `5` | (reserved) | |
| `6` | NOT_ACTIVATABLE | Function configured but cannot be enabled (e.g. speed too low) |
| `7` | FAULT | Speed control system fault |

### Typical byte 0 patterns

| b0 hex | CTRL_TYPE | STATUS | Meaning |
|--------|-----------|--------|---------|
| `0x00` | none | standby | System idle |
| `0x48` | regulator | active | Cruise control holding speed |
| `0x80` | limiter | standby | Limiter configured, inactive |
| `0x88` | limiter | active | Limiter actively capping speed |
| `0xC0` | adaptive | standby | ACC available, not active |
| `0xC8` | adaptive | active | ACC holding gap/speed |

## SET_SPEED (bytes 1-2, idx 1-2)

**Format:** 16-bit unsigned integer (big-endian)  
**Scale:** × 0.01 km/h per LSB  
**Invalid:** `0xFFFF` → no set speed  
**Maximum valid:** `0xFFFE` = 655.34 km/h (practical limit ≤ 250 km/h)

```python
# Decode
speed_raw = (data[1] << 8) | data[2]
if speed_raw == 0xFFFF:
    set_speed_kmh = None  # not set
else:
    set_speed_kmh = speed_raw * 0.01

# Encode
if set_speed_kmh is None:
    speed_raw = 0xFFFF
else:
    speed_raw = min(0xFFFE, round(set_speed_kmh / 0.01))
b1 = (speed_raw >> 8) & 0xFF
b2 = speed_raw & 0xFF
```

### SET_SPEED examples

| km/h | uint16 raw | b1 | b2 |
|------|-----------|----|----|
| 30 km/h | `0x0BB8` | `0x0B` | `0xB8` |
| 90 km/h | `0x2328` | `0x23` | `0x28` |
| 110 km/h | `0x2AF8` | `0x2A` | `0xF8` |
| 130 km/h | `0x32C8` | `0x32` | `0xC8` |
| Not set | `0xFFFF` | `0xFF` | `0xFF` |

## ODOMETER_PARTIAL (bytes 5-7, idx 5-7)

The partial odometer is a resettable trip counter. It is unrelated to the total
odometer in `0x0F6` bytes 2-4.

**Format:** 24-bit unsigned integer (big-endian)  
**Scale:** × 0.001 km per LSB  
**Invalid:** `0xFFFFFF`  
**Maximum:** ~16 777.214 km per trip segment

```python
# Decode
odo_raw = (data[5] << 16) | (data[6] << 8) | data[7]
if odo_raw == 0xFFFFFF:
    partial_odo_km = None  # invalid / not available
else:
    partial_odo_km = odo_raw * 0.001

# Encode
if partial_odo_km is None:
    odo_raw = 0xFFFFFF
else:
    odo_raw = min(0xFFFFFE, round(partial_odo_km / 0.001))
b5 = (odo_raw >> 16) & 0xFF
b6 = (odo_raw >> 8)  & 0xFF
b7 = odo_raw & 0xFF
```

### ODOMETER_PARTIAL examples

| km | uint24 raw | b5 | b6 | b7 |
|----|-----------|----|----|-----|
| 0 km | `0x000000` | `0x00` | `0x00` | `0x00` |
| 0.123 km | `0x00007B` | `0x00` | `0x00` | `0x7B` |
| 1234.567 km | `0x12D687` | `0x12` | `0xD6` | `0x87` |
| Invalid | `0xFFFFFF` | `0xFF` | `0xFF` | `0xFF` |

## Full Frame Decode Snippet

```python
def decode_0x1A8(data: bytes) -> dict:
    """Decode a SPEED_CONTROL (0x1A8) frame."""
    assert len(data) >= 8

    speed_raw = (data[1] << 8) | data[2]
    odo_raw   = (data[5] << 16) | (data[6] << 8) | data[7]

    return {
        'control_type':      (data[0] >> 6) & 0x03,
        'function_status':   (data[0] >> 3) & 0x07,
        'activation_attempt':(data[0] >> 2) & 0x01,
        'unit_mph':           bool((data[0] >> 1) & 0x01),
        'set_speed_kmh':      None if speed_raw == 0xFFFF else speed_raw * 0.01,
        'partial_odo_km':     None if odo_raw == 0xFFFFFF else odo_raw * 0.001,
    }
```

## Simulator Implementation

The `Msg1A8` class in `can_messages.py` encodes and decodes this frame using the
`car.speed_control` (`SpeedControl`) state object.

### `SpeedControl` state fields

| Field | Type | Description |
|-------|------|-------------|
| `control_type` | int (0-3) | `NONE=0`, `REGULATOR=1`, `LIMITER=2`, `ADAPTIVE=3` |
| `function_status` | int (0-7) | `STANDBY=0`, `ACTIVE=1`, … `FAULT=7` |
| `activation_attempt` | int (0-1) | Transient flag |
| `unit_mph` | bool | `False` = km/h (default), `True` = mph |
| `set_speed` | float or None | km/h; `None` encodes as `0xFFFF` |
| `partial_odo` | float or None | km; `None` encodes as `0xFFFFFF` |

### Usage example

```python
from car_state import VirtualCar, SpeedControl

car = VirtualCar()

# Activate cruise control at 110 km/h
car.speed_control.control_type    = SpeedControl.REGULATOR
car.speed_control.function_status = SpeedControl.ACTIVE
car.speed_control.set_speed       = 110.0
car.speed_control.partial_odo     = 87.654  # km since last reset

from can_messages import Msg1A8
frame = Msg1A8().encode(car)
# → [0x48, 0x2A, 0xF8, 0x00, 0x00, 0x00, 0x15, 0x6A]
```

### Default behaviour

When `car.speed_control` is at default values:
- `control_type = NONE`, `function_status = STANDBY` → byte 0 = `0x00`
- `set_speed = None` → bytes 1-2 = `0xFF 0xFF`
- `partial_odo = None` → bytes 5-7 = `0xFF 0xFF 0xFF`

This is the correct "no speed control" idle frame.

## Relationship to 0x0F6 Odometer

| Signal | Frame | Bytes (idx) | Scale | Purpose |
|--------|-------|-------------|-------|---------|
| `ODOMETER` (total) | `0x0F6` | 2-4 | × 0.1 km | Lifetime vehicle km |
| `ODOMETER_PARTIAL` | `0x1A8` | 5-7 | × 0.001 km | Resettable trip counter |

The partial odometer provides three decimal places of precision (1 m resolution),
making it suitable for live trip-computer display.
