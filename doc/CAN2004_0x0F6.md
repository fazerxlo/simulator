# CAN2004 Frame 0x0F6 — BSI_SLOW_DATA (Peugeot 407)

PSA-RE canonical name: `BSI_SLOW_DATA` / `DONNEES_BSI_LENTES`  
Sender: BSI  
Period: 500 ms (real bus); simulator transmits at 100 ms  
Frame length: 8 bytes

## Byte 0 — Status byte

The first byte packs several BSI status flags.  On a real running car the typical value is
`0x88`:

```
0x88 = 1000 1000
       ^^         CONFIG_MODE   = 2 (customer configuration)
         0 0      MAIN_STATUS   = 0 (contact on / normal)
             1    GEN_STATUS    = 1 (generator working)
               00 POWERTRAIN_STATUS = 0
```

The simulator emits `0x88` to match the real-bus pattern.

### Common status byte values

| Value | Meaning |
|-------|---------|
| `0x00` | Factory mode, engine not running |
| `0x08` | Factory mode, generator working |
| `0x88` | Customer config, generator working (normal running) |
| `0x8A` | Customer config, engine starting |
| `0x8B` | Customer config, engine running |

## Coolant Temperature (byte 1, idx 1)

**Formula:** `coolant_°C = raw − 40`  
**Range:** −40 °C (raw `0x00`) to +215 °C (raw `0xFF`)  
**Typical operating range:** 80–105 °C

| Example | Raw | Decoded |
|---------|-----|---------|
| Cold start | `0x08` | −32 °C |
| Warm idle | `0x78` | 80 °C |
| Normal | `0x8C` | 100 °C |
| Overheat alert | `0xC8` | 160 °C |

```python
# Decode
coolant_c = data[1] - 40

# Encode
raw = int(coolant_c + 40)  # clamp to 0x00–0xFF
```

## Odometer (bytes 2-4, idx 2–4)

**Format:** 24-bit unsigned integer (big-endian)  
**Scale:** × 0.1 km per LSB  
**Invalid:** `0xFFFFFF`  
**Maximum:** ~1 677 721.5 km

```python
# Decode
odo_raw = (data[2] << 16) | (data[3] << 8) | data[4]
if odo_raw == 0xFFFFFF:
    odometer_km = None  # invalid / not available
else:
    odometer_km = odo_raw * 0.1

# Encode
if odometer_km is None:
    odo_raw = 0xFFFFFF
else:
    odo_raw = round(odometer_km / 0.1)
b2 = (odo_raw >> 16) & 0xFF
b3 = (odo_raw >> 8) & 0xFF
b4 = odo_raw & 0xFF
```

> **Simulator note:** The bench simulator has no real odometer to emulate.  Bytes 2–4 are
> transmitted as `0xFF 0xFF 0xFF` (invalid sentinel). This is the correct placeholder for
> bench use.

## External Temperature (bytes 5–6, idx 5–6)

**Formula:** `temp_°C = raw × 0.5 − 40`  
**Resolution:** 0.5 °C per LSB  
**Invalid:** `0xFF`  
**Range:** −40 °C (`0x00`) to 87.5 °C (`0xFE`)

Byte 5 (idx 5) is the raw (unfiltered) reading; byte 6 (idx 6) is a low-pass filtered
version that changes more slowly to avoid flicker on the display.

| Example | Raw | Decoded |
|---------|-----|---------|
| Freezing | `0x30` | 4.0 °C |
| Typical summer | `0x84` | 26.0 °C |
| Hot day | `0xB2` | 49.0 °C |
| Invalid | `0xFF` | — |

```python
# Decode
raw = data[5]
if raw == 0xFF:
    ext_temp = None
else:
    ext_temp = raw * 0.5 - 40.0

# Encode
if ext_temp is None:
    raw = 0xFF
else:
    raw = int((ext_temp + 40) * 2)  # clamp 0x00–0xFE
```

## Byte 7 — Reverse, Wipers, Blinkers

```
Bit 7  REVERSE_STATUS     — 1 = reverse gear engaged
Bit 6  FRONT_WIPERS_STATUS — 1 = wipers active (any speed)
Bit 5  WHEEL_POSITION (MSB)
Bit 4  WHEEL_POSITION (LSB)  — combined: 1=RHD, 2=LHD
Bit 3  CLUSTER_INDICATORS_TEST — 1 = lamp-test cycle in progress
Bit 2  (unused)
Bit 1  BLINKERS_STATUS (MSB)
Bit 0  BLINKERS_STATUS (LSB) — combined: 0=none, 1=right, 2=left, 3=both
```

### BLINKERS_STATUS values

| Value | Meaning |
|-------|---------|
| `0` | No blinker active |
| `1` | Right indicator |
| `2` | Left indicator |
| `3` | Both — hazard warning lights |

```python
# Decode
reverse  = (data[7] >> 7) & 1
blinkers = data[7] & 0x03   # 0=none, 1=right, 2=left, 3=both

# Encode
b7 = (reverse << 7) | (blinkers & 0x03)
```

## Full Frame Decode Snippet

```python
def decode_0x0F6(data: bytes) -> dict:
    """Decode a BSI_SLOW_DATA (0x0F6) frame."""
    assert len(data) >= 8

    odo_raw = (data[2] << 16) | (data[3] << 8) | data[4]
    ext_raw = data[5]
    return {
        'status_byte':  data[0],
        'coolant_c':    data[1] - 40,
        'odometer_km':  None if odo_raw == 0xFFFFFF else odo_raw * 0.1,
        'ext_temp_c':   None if ext_raw == 0xFF else ext_raw * 0.5 - 40.0,
        'ext_filt_c':   None if data[6] == 0xFF else data[6] * 0.5 - 40.0,
        'reverse':      bool((data[7] >> 7) & 1),
        'wipers':       bool((data[7] >> 6) & 1),
        'blinkers':     data[7] & 0x03,   # 0=none 1=right 2=left 3=both
    }
```

## Simulator Implementation

State fields driven by this frame:

| `car.bsi` field | Source byte/bits | Default |
|-----------------|-----------------|---------|
| `coolant` | byte 1 (idx 1), raw − 40 | `0` |
| `temperature` | byte 5 (idx 5), raw × 0.5 − 40 | `20` |
| `reverse` | byte 7 (idx 7) bit 7 | `0` |
| `blinkers` | byte 7 (idx 7) bits 1-0 | `0` |

Byte 7 bit 6 (`FRONT_WIPERS_STATUS`) and bits 5-4 (`WHEEL_POSITION`) are decoded
by the frame encoder but not yet stored as dedicated state fields; they are forwarded
from the raw byte.

### Bench encoding notes

- Byte 0 is fixed at `0x88` (customer config + generator working).
- Odometer bytes 2-4 are `0xFF 0xFF 0xFF` (no real odometer on the bench).
- Bytes 5 and 6 (ext temp and filtered ext temp) both encode the same
  `car.bsi.temperature` value.
